
*객체(Object)
현실세계에 존재하는 유형(구체적), 무형(추상적)의 실체(개념)
객체의 예시 : 인간, 자동차, 강의, 축구
객체가 가지는 것: 상태(State=Field), 동작(Behavior=Method)
자동차의 상태 : 색상, 현재의 기어, 현재의 속도, 연식 등...
자동차의 동작 : 시동, 출발, 멈춤, 감속, 가속 등...

*객체지향 용어
클래스: 객체 지향 언어에서 프로그램을 개발하는 단위, 객체를 정의하는 틀 또는 설계도
필드: 객체의 특성(변수), 클래스의 3요소
메서드: 수행할 작업을 나열한 코드의 모임 (Method), 클래스의 3요소
생성자: new 메서드로 호출되는 특별한 메소드, 클래스의 3요소
캡슐화: 외부에 드러나지 않게 숨기는 것
다형성:
추상화:

*변수
Variable
기억 공간에 데이터를 보관, 기억공간을 변수라 하며 데이터 타입으로 구분

*상수
고정된 값을 가진 자료.
선언: final double PI = 5.2;
조건: 상수는 암묵적으로 대문자로 만든다.
주의: 상수 선언할 때 값을 꼭 넣어줘라, 선언 이후 바꿀 수가 없다.

*데이터형=자료형
int, double, char, String(순자, 문자, 문자열...)

*배열
선언1: int [] num = new int[5];
선언2: int [] num = {10, 20, 30};
num.length: 배열의 크기

*메소드
조건: 암묵적으로 두개의 단어 사용시 getAge, setAge

*메소드 오버로딩(메소드 중복정의)
반환타입 메소드이름(매개변수){}
조건: 메소드 이름이 같아도 매개변수가 하나라도 다르면(개수가 다르거나, 자료형이 다르면) 같은 이름 사용 가능(이름 중복 허용)
주의: 반환타입과는 무관, 매개변수명과는 무관

*생성자
생성자가 없으면 컴파일러가 빈껍데기 생성자를 만든다.
조건: 객체를 초기화하는 "특수메소드", 메소드이름은 클래스이름으로 필수
주의: 반환타입 없음, 객체 생성시 new 연산자 필요, 오버로딩 가능

*this 키워드
this.변수명 = 자기 자신의 변수명을 지정함
조건: 지역과 전역의 변수명이 같으면 (this.변수명 == 전역, 변수명 == 지역)
주의: 전역과 지역의 변수명 중복 없으면 this 필요없음

*접근제어
조건: private는 클래스 내에서만 접근, public은 모두 접근 가능
키워드: public, private, protected, default
캡슐화: 외부에 드러나지 않게 숨기는 것

*상속
부모에게서 확장
부모의 필드, 메소드를 기본적으로 가지는 클래스를 만든다.
자식만의 필드와 메소드를 가질 수도 있다.
선언: public void 클래스명 extends 부모클래스명

*자식 생성자
super(); 자식 생성자에서 부모 생성자를 호출하는 키워드.
조건1 : 자식 생성자 실행 초기에 적어야 한다.
조건2 : 부모클래스의 생성자 매개인수가 있을 때 사용하고 super(매개인자...); 이렇게 사용해야 됨.
조건3 : 자식의 매개인자가 있는데 부모의 매개인자가 없을수는 없음.

*오버라이딩
부모 클래스의 메소드를 자식 클래스에서 재정의
(같은 이름인데 다른 기능)
(자식이 부모랑 똑같은 메소드 이름을 가지지만 내부가 다름)

*오버로딩
메서드의 기능이 다르지만 같은 이름을 쓸 수 있도록 하는 것
절차지향에선 함수 이름이 프로그램 내에서 유일하여야 한다.
조건1: 매개인자 자료형의 개수가 다르면 됨.
조건2: 매개인자 자료형의 개수가 같아도 매개인자 자료형이 다르면 됨
주의: 매개인자 이름이 다른건 상관없음.

*super 키워드
부모의 오버라이딩 한 메소드를 명시적으로 호출 또는
부모의 값을 불러옴

*예외
프로그램 실행 중에 발생한 오류
ArithmeticException 0으로 나눌 수 없음
IllegalArgumentException 메서드에 부적절 매개인수 전달
IndexOutOfBoundsException 배열 또는 벡터 범위 벗어남
NoSuchElementException 요구한 원소 없음
NullPointerException null값을 가진 변수 접근 시도
NumberFormatException 숫자로 못바꾸는 문자열 변환 시도
